第1章 C#和.NET框架

C#编程语言是为在巨硬公司的.NET框架上开发程序而设计的。在开始之前，首先要指出C#的正确发音：see sharp。

1 .NET 框架的组成

.NET 框架由三部分组成：编程工具、BCL（Base Class Library，基类库）和执行环境CLR（Common Language Runtime，公共语言运行时）。CLR在运行时管理程序的执行，包括以下内容：

  • 内存管理和垃圾收集。
  • 代码安全验证。
  • 代码执行、线程管理及异常处理。

1.1 基类库（BCL）

.NET 框架提供了一个庞大的基础类库，很自然的，它被称为基类库（Base Class Library，BCL）。（有时称为框架类库—Framework Class Library，FCL。）在编写程序时，可以使用以下的类：

  • 通用基础类：这些类提供一组极为强大的工具，可以应用到许多编程任务中，比如文件操作、字符串操作、安全和加密。
  • 集合类：这些类实现了列表、字典、散列表以及位数组。
  • 线程和同步类：这些类用于创建多线程程序。
  • XML 类：这些类用于创建、读取以及操作XML文档。

1.2 编译成 CIL

.NET 语言的编译器接受源代码文件，并生成名为程序集的输出文件。

  • 程序集要么是可执行的，要么是DLL。
  • 程序集里的代码并不是本机代码，而是一种名称为CIL（Common Intermediate Language，公共中间语言）的中间语言。
  • 程序集包含的信息中，包括下列项目：
    • 程序的CIL；
    • 程序中使用的类型的元数据；
    • 对其他程序集引用的数据。

1.3 编译成本机代码并执行

程序的 CIL 直到被调用运行时才会被编译成本机代码。在运行时，CLR 执行如下步骤：

（1）检查程序集的安全特性；

（2）在内存中分配空间；

（3）把程序集中的可执行代码发送给即时（just-in-time，JIT）编译器，把其中的一部分编译成本机代码。

程序集中的可执行代码只在需要的时候由JIT编译器编译，然后它就被缓存起来以备在后来的程序中执行。使用这个方法意味着不被调用的代码不会被编译成本机代码，而且被调用到的代码只被编译一次。

一旦 CIL 被编译成本机代码，CLR就在它运行时管理它，执行如释放无主内存，检查数组边界、检查参数类型和管理异常之类的任务。两个重要的属于由此而生：

  • 托管代码：为.NET 框架编写的代码称为托管代码（managed code），需要CLR。
  • 非托管代码：不在CLR控制之下运行的代码，比如Win32 C/C++ DLL，称为非托管代码（unmanaged code）。

1.4 CLR

.NET 框架的核心组件是 CLR，它在操作系统的顶层，负责管理程序的执行，提供下列服务：

  • 内存管理；
  • 异常处理；
  • 垃圾收集；
  • 反射服务；
  • JIT编译器；
  • 类加载器；
  • 安全服务。

1.5 CLI

CLI（Common Language Infrastructure，公共语言基础结构）把.NET框架的所有组件连接成一个内聚的、一致的系统。它展示了系统的概念和架构，并详细说明了所有软件都必须遵守的规则和约定。

CLI 有如下两个重要组成部分：

  • CTS（Common Type System，公共类型系统）：定义了那些在托管代码中一定会使用的类型的特征，可以确保系统类型和用户定义类型能够被任何.NET兼容的语言所使用。

  • CLS（Common Language Specification，公共语言规范）：详细说明了一个.NET兼容编程语言的规则、属性和行为，其主题包括数据类型、类结构和参数传递。
